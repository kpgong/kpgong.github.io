看毛片算法

2021年03月16日 6 min read # 字符串 # 算法

不是，你们都不看的吗？

—— 某 V 老嗨

问题

现有两个字符串 A 和 B，要判断 B 是否是 A 的子串。这个问题叫作模式匹配。可能是子串的串（B）叫模式串，可能包含子串的串（A）叫主串。假设 A、B 的长度分别为 M,NM,NM,N。

简单的模式匹配

暴力：

// Returns the index where the first character of sub appears in main
// if sub is a sub-string of main,
// or -1 otherwise.
int search(string main, string sub)
{
#define in(ch, str) ((ch) < (str).size())
    for (int i = 0; i < main.size(); i++)
    {
        int j = i, k = 0;
        while (in(j, main) && in(k, sub) && main[j] == sub[k])
        {
            ++j;
            ++k;
        }
        if (j == sub.size()) return i;
    }
    return -1;
}

时间复杂度为 O(MN)\Omicron\left(MN\right)O(MN)。太复杂了！（迫真）

特殊的情况

A = "abaabbababa"
B = "ababa"

按照上面的暴力方法，当 i == 0, j == 3 时，发现 A[3] != B[3]，下一步应该是令i == 1, j == 0，然后比较 A[1] 与 B[0]。

      i
      j
A = "abaabbababa"
B =  "ababa"
      k

可以将上面代码中的 i 看作字符串 B 与字符串 A 的“相对位置”。当 i == 0 时，A 中的第 kkk 个字符对齐于 B 中的第 kkk 个字符；当 i == 1 时，A 中的第 kkk 个字符对齐于 B 中的第 (k+1)(k + 1)(k+1) 个字符；以此类推。每次比较字符，比较的都是 A[j] 和 B[k]。这就是双指针法。

在 i == 0 时我们就已经访问过 A[1] 和 B[0] 且知道 B[0] != A[1]，没必要再一次比较，甚至可以直接跳过 i == 1 的情况——因为当“相对位置”为 111 时，第一个对齐的字符都不一样。

因为当 i == 0 时，我们已经知道 A[2] == 'a' 且 B[0] == 'a'，不妨直接快进到下面的情况：让 A[2] 和 B[0] 对齐，然后直接比较 A[3] 和 B[1]，即 i == 2, j == 3, k == 1。这样一来就跳过了好几次重复的比较。而且，相比于上面的暴力方法，指针 j 没有回头。这样的“快进”是利用了先前比较时得到的信息。

       ij
A = "abaabbababa"
B =   "ababa"
        k

不难发现，如果 B 的子串 B[0..k-1] 有相同的前缀和后缀，那么可以“快进”：跳过某些比较且指针 j 不需要回溯。

KMP 算法

考虑 B[k] 未能成功匹配（下称“失配”）的情况，如果模式串已经匹配的部分有相同的前缀和后缀，假设最长的相同的前缀和后缀长度为 sss。那么主串的指针不用移动，将模式串上的指针退回到 B[s] 处重新比较即可。如果没有相同的前缀和后缀，那么应该将主串的指针向前移动一个位置，然后将模式串的指针退回到 B[0] 处。

模式串上的指针每次退回时，退回的位置都只与模式串本身有关，所以可以预先处理记下退回位置。

以上就是 KMP 算法的核心思想。如果模式串有很多重复出现的部分，那么 KMP 算法会很快；如果模式串中根本没有重复出现的部分，那么 KMP 算法与暴力算法相同。

我们把 B[k] 失配后的退回位置记为 next[k]，则有

next[k]={−1,k=0;已匹配部分中最长的相同前后缀的长度,k>0.next[k] =\left\{\begin{aligned} &-1, k = 0;\\ &已匹配部分中最长的相同前后缀的长度,k > 0. \end{aligned}\right. next[k]={​−1,k=0;已匹配部分中最长的相同前后缀的长度,k>0.​

指针不能退回到 −1-1−1 处，这里的 −1-1−1 表示要将主串的指针前移并将模式串的指针移到索引 000 处。请注意：上面提到的串的“前缀”及“后缀”均不是串本身，所以 next[k]≤k−1next[k] \le k - 1next[k]≤k−1。

数组 next 叫部分匹配表（Partial Match Table, PMT）。

代码如下

#include <bits/stdc++.h>
using namespace std;
vector<int> getNext(string str)
{
    vector<int> ret(str.size());
    for (int i = 1; i < str.size(); i++)
    {
        int k = ret[i - 1];
        while (str[k] != str[i - 1])
        {
            if (k == 0)
            {
                k = -1;
                break;
            }
            k = ret[k];
        }
        ret[i] = k + 1;
        // Here, a string is not a prefix or a suffix of itself.
        // So, next[i] is strictly less than i.
        if (ret[i] == i) --ret[i];
    }
    ret[0] = -1;
    return ret;
}
int search(string a, string b)
{
    auto next = getNext(b);
    int i = 0, j = 0;
    while (i < a.size())
    {
        while (a[i] != b[j] && i < a.size())
        {
            j = next[j];
            if (j < 0)
            {
                ++i;
                j = 0;
            }
        }
        ++j, ++i;
        if (j == b.size())
            return i - b.size();
    }
    return -1;
}
int main()
{
    string a, b;
    cin >> a >> b;
    cout << search(a, b) << endl;
    return 0;
}
