解析1

七星的难度确实高。我将按照子任务的提示，一步一步发散。下面先解决子任务1。

很容易想到动态规划（实际上只是单纯的递推）。如果我们能凑出 BBB 的倍数，那么这个数除以 BBB 的余数应该为 000。以输入样例1为例，首先考察一位数的情况。

输入样例1：

3 7 3
1 4 9

可选的数字有 1 4 9。由

1÷7=0⋯⋯14÷7=0⋯⋯49÷7=1⋯⋯21\div 7=0\cdots\cdots 1\\ 4\div 7=0\cdots\cdots 4\\ 9\div 7=1\cdots\cdots 2 1÷7=0⋯⋯14÷7=0⋯⋯49÷7=1⋯⋯2

没有 7 的倍数。下面考察两位数的情况。当个位数为 111 ，有

11÷7=1⋯⋯414÷7=2⋯⋯019÷7=2⋯⋯511\div 7=1\cdots\cdots 4\\ 14\div 7=2\cdots\cdots 0\\ 19\div 7=2\cdots\cdots 5 11÷7=1⋯⋯414÷7=2⋯⋯019÷7=2⋯⋯5

在计算11、14、19的余数的时候，没必要算除法，可以利用之前计算过的 1 mod 7=11\bmod 7= 11mod7=1 来得出11、14、19的余数，即

11 mod 7=(1×10+1) mod 7=(1 mod 7)×(10 mod 7)+(1 mod 7)\begin{aligned} 11\bmod 7&=(1\times 10 + 1)\bmod 7\\ &=(1\bmod 7) \times (10\bmod 7) + (1\bmod 7) \end{aligned} 11mod7​=(1×10+1)mod7=(1mod7)×(10mod7)+(1mod7)​

也就是说，位数为 iii 时的余数可以利用位数为 (i−1)(i-1)(i−1) 时的余数求出。

设 xxx 为数字 c1,c2,⋯ ,cKc_1, c_2, \cdots,c_Kc1​,c2​,⋯,cK​ 组成的正整数，定义下述二维数组：

dp[x的位数][x除B的余数]=满足条件的x的数量dp\left[x的位数\right]\left[x除 B 的余数\right]=满足条件的x的数量 dp[x的位数][x除B的余数]=满足条件的x的数量

则有转移方程

dp[n][r]=∑(10t+ci)≡r(modB)dp[n−1][t]dp\left[n\right]\left[r\right]=\sum_{\left(10t+c_i\right)\equiv r\pmod B}{dp\left[n-1\right]\left[t\right]} dp[n][r]=(10t+ci​)≡r(modB)∑​dp[n−1][t]

这样做的时间复杂度是 O(NB)\Omicron\left(NB\right)O(NB)，空间复杂度是 O(NB)\Omicron\left(NB\right)O(NB)。你能把空间复杂度优化到 O(B)\Omicron\left(B\right)O(B) 吗？

代码1

struct Solution1
{
#define sfor(i, n) for (int i = 1; i <= (n); ++i)
#define tfor(i, n) for (int i = 0; i < (n); ++i)
    using int64 = long long;
    static constexpr int M = 1000000007;

    vector<int> c;
    int64 k, n, b;
    vector<vector<int> > dp;

    Solution1()
    {
        cin >> n >> b >> k;
        c = vector<int>(k);
        tfor(i, k) cin >> c[i];

        dp = vector<vector<int> >(n + 1);
        for (auto& v : dp) v = vector<int>(b);
        dp[0][0] = 1;
        sfor(i, n) {
            for (auto& j : c) {
                tfor(k, b) {
                    auto& t = dp[i][(j + 10 * k) % b];
                    t = (t + dp[i - 1][k]) % M;
                }
            }
        }
        cout << dp[n][0] << endl;
    }
#undef tfor
#undef sfor
};

int main()
{
    Solution1();
    return 0;
}

解析2

解析1中的转移方程表明，dp[n]dp[n]dp[n] 只与 dp[n−1]dp[n-1]dp[n−1] 有关，而 dp[n−2],dp[n−3]dp[n-2],dp[n-3]dp[n−2],dp[n−3] 等都可以舍弃。所以二维数组 dp 只需要 222 行就足够用了。

下面解决子任务2。

将 dp[i]dp[i]dp[i] 视为 BBB 行的列向量，则有

A⋅dp[i−1]=dp[i]A\cdot dp[i-1]=dp[i] A⋅dp[i−1]=dp[i]

其中，AAA 是一个与 iii 无关的 BBB 阶方阵。想一想为什么。

要输出的结果只有 dp[n][0]dp[n][0]dp[n][0]，其他的值我们并不关心，则有

dp[n]=An⋅dp[0]dp[n]=A^{n}\cdot dp[0] dp[n]=An⋅dp[0]

矩阵快速幂！

实际上，如果转移方程是线性方程组、其系数又不随转移过程而更改，那么转移方程都可以写成类似的矩阵乘法形式，从而使用矩阵快速幂求解。

这样做的时间复杂度是 O(B3log⁡n)\Omicron\left(B^3\log{n}\right)O(B3logn)。
